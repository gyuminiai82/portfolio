# 제품 요구사항 정의서 (PRD): React + Tailwind CSS 포트폴리오

## 1. 프로젝트 개요
* **프로젝트명:** 개발자 여정 포트폴리오 랜딩 페이지 (The Developer's Journey)
* **목적:** 2005년부터 현재까지 쌓아온 풍부한 개발 경력을 인터랙티브한 타임라인으로 시각화하여, 방문자(채용 담당자)에게 깊은 인상을 남기고 프론트엔드 구현 능력을 증명함.
* **핵심 컨셉:** 사용자가 스크롤을 내릴 때 화면 중앙의 가이드를 따라 사람 발자국이 순차적으로 찍히며 해당 연도의 경력 카드가 활성화되는 스크롤 스파이(Scroll Spy) 시스템.

## 2. 기술 스택
* **Framework / 빌드 도구:** React (Vite) — 가볍고 빠른 개발 및 빌드 환경 제공
* **Styling:** Tailwind CSS — 유틸리티 클래스를 활용한 빠른 레이아웃 및 반응형 디자인
* **Interactions:** Vanilla JS (Intersection Observer API, 표준 Scroll Event) — 외부 애니메이션 라이브러리 없이 순수 웹 표준 기술로 성능 최적화

## 3. 핵심 기능 및 화면 구성

### 3.1. 스크롤 스파이 & 발자국 인터랙션
* **중앙 타임라인:** 화면 중앙을 관통하는 수직선(또는 점선)을 배치합니다.
* **발자국 연출 (Scroll Trigger):** 
    * 각 경력 카드 옆에 미리 왼발/오른발 SVG 아이콘을 지그재그로 배치해 둡니다.
    * 사용자가 스크롤을 내릴 때, 스크롤 위치가 특정 경력 섹션 영역에 도달하면 `IntersectionObserver`가 이를 감지합니다.
    * 감지된 순간 해당 발자국의 Tailwind 클래스를 `opacity-0`에서 `opacity-100`으로 전환하고, `duration-500`과 같은 트랜지션을 주어 발자국이 자연스럽게 나타나도록 만듭니다.
* **카드 활성화:** 발자국이 활성화됨과 동시에 해당 경력 카드가 투명도와 위치 플러그인 효과(`opacity-100 translate-y-0`)를 받으며 부드럽게 등장합니다.

### 3.2. 반응형 타임라인 레이아웃
* **데스크톱 (md 이상):** 
    * 중앙 라인을 기준으로 좌측과 우측에 번갈아가며 경력 카드가 배치되는 지그재그(Zig-Zag) 형태.
    * 발자국 역시 왼쪽, 오른쪽 교차하며 찍히는 연출.
* **모바일 (sm 이하):**
    * 화면 폭이 좁으므로 수직 라인을 좌측 가장자리로 옮기고, 모든 경력 카드가 우측 한 열(Single Column)로 정렬되는 형태로 자동 전환.
    * 발자국 애니메이션은 간소화하여 수직선을 따라 아래로만 찍히도록 처리.

### 3.3. 네비게이션 인디케이터
* 화면 우측 또는 좌측 상단에 현재 보고 있는 회사/연도를 표시하는 미니 플로팅 바 배치.
* 특정 회사를 클릭하면 `window.scrollTo({ top: ..., behavior: 'smooth' })`를 통해 해당 섹션으로 부드럽게 이동.

## 5. 단계별 개발 계획 (Milestone)

* **1단계: 프로젝트 뼈대 및 데이터 세팅**
  * Vite로 React 프로젝트 생성 및 Tailwind CSS 환경 설정.
  * 상기 경력 데이터를 `careerData.js` 파일에 객체 배열 형태로 정리.
* **2단계: 반응형 타임라인 UI 마크업**
  * 수직 가이드라인을 중심으로 데스크톱 해상도에서의 지그재그 배치 구현.
  * 모바일 화면 크기 변화에 맞춰 일렬로 정렬되는 미디어 쿼리(Tailwind의 `md:`, `flex-row-reverse` 등) 적용.
* **3단계: 스크롤 스파이 컴포넌트 구현 (핵심)**
  * 개별 경력 카드를 래핑하는 `TimelineItem` 컴포넌트 작성.
  * 브라우저 기본 `IntersectionObserver`를 활용해 카드가 화면(Viewport)에 30~50% 이상 들어왔을 때 상태 변수를 업데이트하는 커스텀 훅 설계.
  * 활성화 상태에 맞춰 발자국 SVG 노출 및 카드의 애니메이션 효과 바인딩.
* **4단계: 마무리 최적화 및 정적 배포**
  * 스크롤 시 부자연스러운 레이아웃 흔들림 방지 및 하드웨어 가속(`will-change`) 적용.
  * GitHub Pages, Vercel 등을 활용한 무료 정적 호스팅 배포 연동.
